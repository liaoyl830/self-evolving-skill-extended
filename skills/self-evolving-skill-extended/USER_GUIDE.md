## 🚀 快速开始

### 什么是 sessions_yield?

sessions_yield 是一个**异步复盘系统**，让你无需等待就能获得复盘结果。

**传统方式:**
```
用户：复盘
小斗：[思考 30 秒...]
小斗：✅ 完成
```

**sessions_yield 方式:**
```
用户：复盘
小斗：🫡 收到，开始复盘今日任务... (1 秒内回复)
[后台自动处理 30 秒]
小斗：✅ 完成通知 (主动推送)
```

## 🔧 高级功能

### 1. 异步 vs 同步

**异步模式 (默认):**
- ✅ 立即响应 (<1 秒)
- ✅ 后台执行
- ✅ 主动通知
- ⚠️ 需要等待完成通知

**同步模式:**
- ✅ 立即获得完整结果
- ⚠️ 需要等待 30-40 秒
- ⚠️ 阻塞式体验

**选择模式:**
```
异步复盘  # 默认
同步复盘  # 立即获得结果
```

---

### 2. 通知设置

**通知类型:**
- 控制台通知 (总是启用)
- 系统事件通知
- 文件通知
- 超时警告 (>5 分钟)
- 错误通知

**查看未读通知:**
```
查看未读通知
```

**标记为已读:**
```
标记通知为已读
```

---

### 3. 错误处理

**自动重试:**
- 失败后自动重试 (最多 3 次)
- 重试间隔：1 秒

**智能降级:**
```
异步失败 → 重试 (3 次) → 同步模式 → 完成
```

**错误日志:**
- 位置：`error_logs/*.log`
- 自动记录所有错误
- 包含详细堆栈信息

---

### 4. 后台任务管理

**查看后台任务:**
```
查看后台任务
```

**任务状态:**
- `running` - 运行中
- `completed` - 已完成
- `failed` - 失败
- `timeout` - 超时

**超时处理:**
- 默认超时：5 分钟
- 超时后自动发送警告
- 可选择重试或取消

## ❓ 常见问题

### Q1: 为什么选择异步模式？

**A:** 异步模式让你无需等待，可以继续使用其他功能。适合：
- 日常复盘
- 长时间任务
- 多任务并行

### Q2: 异步模式多久能完成？

**A:** 通常 30-40 秒。完成后会主动通知你。

### Q3: 如果异步失败怎么办？

**A:** 系统会自动：
1. 重试 (最多 3 次)
2. 降级到同步模式
3. 发送错误通知

你仍然能获得结果，只是方式不同。

### Q4: 如何查看历史复盘？

**A:** 
```
查看历史复盘
查看上个月的复盘报告
```

或手动查看文件：
```
memory/review-*.md
```

### Q5: 复盘报告保存在哪里？

**A:** 
- 每日报告：`memory/review-YYYY-MM-DD.md`
- 长期记忆：`MEMORY.md`
- 通知文件：`notifications/*.json`

### Q6: 可以同时进行多个复盘吗？

**A:** 可以。系统支持并发多个后台任务。

### Q7: 如何关闭异步模式？

**A:** 
```
使用同步模式
```

或直接说：
```
同步复盘
```
