# 安全说明 / Security Notice

## ⚠️ 安全使用规范

### 代码安全
- 🔒 不要在代码中写入密码、API Key、密钥等敏感信息
- 🔒 使用 `.env` 文件存储敏感配置，并添加到 `.gitignore`
- 🔒 第三方技能需经过代码审查方可使用
- 🔒 所有文件统一使用 UTF-8 编码，防止乱码导致的安全问题

### 执行安全
- ⚠️ 智能体执行物理硬件操作时必须人工确认
- ⚠️ 不要在生产环境直接使用未测试的技能
- ⚠️ 敏感操作（删除文件、外部消息等）需设置审批流程
- ⚠️ 定期审计 Git 历史记录和飞书操作记录

### 数据安全
- 🔐 不要将个人数据、隐私信息提交到公开仓库
- 🔐 飞书 Token、GitHub Token 等仅存储在本地 `.env` 文件
- 🔐 发现敏感信息泄露应立即撤销相关授权并更新 Token

---

## 🛡️ 防护措施

### 权限管理
| 权限类型 | 建议设置 |
|---------|---------|
| 工作区文件 | ✅ 允许读写 |
| Git 操作 | ✅ 允许提交推送 |
| 外部 API | ⚠️ 需配置 Token |
| 系统命令 | ⚠️ 受限执行 |
| 敏感操作 | ❌ 需人工确认 |

### 审计追踪
- ✅ 所有 Git 操作有提交记录
- ✅ 所有飞书操作有消息记录
- ✅ 所有文件修改可追溯
- ✅ 定期审查 `memory/` 目录内容

---

## 🐛 漏洞反馈

发现安全漏洞？请按以下方式反馈：

1. **不要**在公开 Issue 中披露漏洞细节
2. 通过飞书私信联系项目维护者
3. 提供漏洞描述、复现步骤、影响范围
4. 等待修复后验证

---

## 📋 安全检查清单

### 提交前自查
- [ ] 无敏感信息（Token、密码、密钥）
- [ ] 无个人隐私数据
- [ ] 文件编码 UTF-8
- [ ] `.env` 文件已添加到 `.gitignore`

### 定期审查
- [ ] 审查 Git 历史中的敏感信息
- [ ] 检查飞书授权范围
- [ ] 更新过期的 Token
- [ ] 清理不再需要的权限
